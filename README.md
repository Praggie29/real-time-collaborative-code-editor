# Real-Time Collaborative Code Editor

A real-time collaborative code editor built with **React**, **Monaco Editor**, **Y.js**, **Socket.IO**, and **Express**. Multiple users can join a shared editing session, see each other's changes in real time, and collaborate on code simultaneously.

🌐 **Live Demo**: [http://docker-aws-project-lb-1334929143.ap-northeast-1.elb.amazonaws.com/](http://docker-aws-project-lb-1334929143.ap-northeast-1.elb.amazonaws.com/)

---

## Features

- **Real-Time Collaboration** – Powered by **Y.js** (CRDT) and **Socket.IO**, every keystroke is synchronised across all connected clients.
- **Monaco Editor** – The same editor that powers VS Code, with syntax highlighting, themes, and language support (JavaScript by default).
- **Live User List** – See who else is currently editing in the sidebar.
- **Username-based Sessions** – Enter a username to join a shared document; all users with the same URL share the same editor state.
- **Health Check Endpoint** – `GET /health` for load balancer and monitoring integration.
- **Dockerised & Cloud Ready** – Multi-stage Docker build, AWS ECS / Fargate deployment support, and CI/CD pipeline configuration.

## Architecture

```
┌──────────────┐     Socket.IO      ┌─────────────────┐
│   Browser A   │ ◄──────────────►   │                 │
│  (Y.Doc +     │                    │   Express       │
│   Monaco)     │                    │   + Socket.IO   │
└──────────────┘                     │   + Y-SocketIO  │
                                     │                 │
┌──────────────┐     Socket.IO      │   (Server)      │
│   Browser B   │ ◄──────────────►   │                 │
│  (Y.Doc +     │                    └───────┬─────────┘
│   Monaco)     │                            │
└──────────────┘                             │
                                     ┌───────▼─────────┐
                                     │   Public /       │
                                     │  (Vite-built     │
                                     │   static files)  │
                                     └─────────────────┘
```

- Each client creates a local **Y.js document (Y.Doc)** with a shared text type (`yText`).
- A **Socket.IO provider** (`y-socket.io`) synchronises the Y.Doc across all clients via the server.
- The **Monaco editor** is bound to `yText` using **y-monaco** so all edits propagate in real time.
- The server is stateless; Y.js handles conflict resolution through its **CRDT** algorithm.

## Getting Started

### Prerequisites

- Node.js >= 18
- npm

### 1. Clone the Repository

```bash
git clone https://github.com/Praggie29/real-time-collaborative-code-editor.git
cd collaborative-editor
```

### 2. Run the Backend

```bash
cd Backend
npm install
npm run dev        # starts on http://localhost:3000
```

### 3. Run the Frontend (Development)

Open a **new terminal**:

```bash
cd Frontend
npm install
npm run dev        # starts on http://localhost:5173
```

> **Note**: The `Frontend/vite.config.js` includes a proxy so API/Socket.IO requests are forwarded to `localhost:3000`.

### 4. Open in Browser

- Navigate to [http://localhost:5173](http://localhost:5173)
- Enter a username and click **Join**
- Open the same URL in another browser/tab with a different username to see collaboration in action!

## Deployment

This project is deployed on **AWS ECS (Fargate)** behind an **Application Load Balancer**.

### CI/CD Pipeline

The `aws/` directory contains all necessary files:

- **`buildspec.yml`** – AWS CodeBuild instructions (builds Docker image, pushes to ECR).
- **`appspec.yml`** – AWS CodeDeploy configuration for ECS blue/green or rolling deployment.
- **`task-definition.json`** – ECS task definition (CPU, memory, port mappings, container definitions).

### Environment Variables (Backend)

| Variable       | Default | Description                         |
|----------------|---------|-------------------------------------|
| `PORT`         | `3000`  | Server port                         |
| `CORS_ORIGIN`  | `*`     | Allowed CORS origin for Socket.IO   |

  "success": true,
  "uptime": 123.45
}

## How It Works

1. **User joins** by entering a username. The app creates a Y.js document and connects to the server via Socket.IO.
2. **Y-SocketIO** synchronises the document state across all connected clients. Any edit is immediately propagated.
3. **Monaco editor** is bound to Yjs's shared text type (`yText`) via `y-monaco`. Changes from other users appear in real time.
4. **Awareness protocol** broadcasts username and presence information so the sidebar shows active users.
5. **Graceful shutdown** – The server listens for `SIGTERM`/`SIGINT` to close connections cleanly (important for container orchestration).
